Objectif(s) technique(s)

Mettre en place une architecture microservice polyglotte à l'aide d'un Docker Swarm.

Note : Une section ressources est présente, pour vous guider dans vos recherches et vos implémentations.

Architecture(s)

(overview_architecture.jpg "Overview-Architecture").

Infrastructure

Chaque composant de l'architecture devra être conteneurisé. Afin de créer un écosystème de conteneur, un Docker Swarm devra être mis en place. L'infrastructure devra avoir un système de service discovery pour la gestion des services.

Logicielle

Tous les services devront mettre en place une architecture hexagonale; La qualité de code devra correspondre à minimat aux principes SOLID et à l'implémentation de patron de conception (si nécessaire).

Architecture Hexagonale

Une architecture hexagonale a pour objectif, de mettre une abstraction entre l'objet métier (domain) et les parties de l'application qui l'exploitent (ex : api web et persistence).

Pour se faire, on met en places des ports , il en existe deux sortes : - Sortant (SPI) : ils permettent au domain d'interagir avec l'extérieur, comme les bases de données ou les services externes. - Entrant (API) : ils permettent à l'extérieur d'interagir avec le domain, comme les API ou les interfaces utilisateur.

Ces ports seront utilisés par des adapters, il en existe deux types : - Primaire : ils utilisent les ports entrant pour interargir avec le domain. - Secondaire : ils implémentent les ports sortant pour interargir avec le domain.
