HACKATHON

Intervenant : MARIN Geoffrey

Sujet

Mise en oeuvre d'une architecture microservices polyglotte conteneurisée pour répondre au besoin applicatif d'un CE d'entreprise.

Contrainte(s)

L'équipe du projet sera formé de 4 groupes de 4 personnes, chaque groupe :

Le projet est commun à tous les groupes; chaque groupe sera en charge d'une ou plusieurs parties de l'application.

Exemple de découpage d'application en plusieurs parties : IHM (Frontend), API Utilisateur (Backend #1), API Tâches (Backend #2), API Tableaux (Backend #3), Base de données, DevOps (CI/CD, Docker), Documentation (mkdocs, javadoc, swagger, etc...), Présentation (canvas, ppt, etc...), Recette (test).

Le choix des technologies de développement est globalement libre mais adapté et justifié.

Technologie(s)/Outil(s) imposé(e)(s): - Docker/Docker Swarm - GitHub

Proposition de technologie(s) : - Documentation : MkDocs (MarkDown)
